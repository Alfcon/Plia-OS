Getting Started

Plia-OS is a local AI assistant with voice, memory, web search, calendar, home automation, and more — all running on your own hardware.

1. Install

Run once after cloning the repo:

  • python -m venv .venv && source .venv/bin/activate
  • pip install -r requirements.txt && pip install -e . --no-deps

2. Start

  • source .venv/bin/activate
  • python core/main.py
  • Dashboard opens at http://localhost:8000

3. First Steps

  • Text chat — type in the box at the bottom and press Send. Works immediately, no setup needed.
  • Voice — requires a microphone. Go to Settings → Voice to pick a wake word model and TTS engine. Say the wake word to start listening.
  • Memory — say "Remember that…" and Plia stores the fact for future conversations.
  • Web search — say "Search for…" and Plia fetches results via DuckDuckGo by default (no key needed).

4. Optional Extras

  • Google Search — Docs → Web for setup.
  • Google Calendar — Docs → Calendar for setup.
  • Home Assistant — Docs → Home for setup.
  • Cloud LLM fallback — pip install -e ".[fallback-openai]" or ".[fallback-anthropic]", then configure in Settings → LLM.
  • Dramabox TTS — pip install -e ".[dramabox]"
  • Playwright scraping — pip install -e ".[playwright]"

5. Key Concepts

  • Settings — configure voice, LLM, integrations. All changes persist to ~/.plia/config.json.
  • Modules — Python files in modules/ that add tools. Drop a file in and restart to add new capabilities.
  • Agents — specialist handlers (memory, web, code, calendar, home, reminder). The supervisor routes your message to the right one automatically.
  • Commands — see Docs → Commands for the full list of trigger phrases.
  • Tools — see Docs → Tools for all 47 functions the LLM can call.

Voice — Setup & Usage

Wake Word

Plia-OS listens for a wake word before activating the microphone.

  • Go to Settings → Voice to select a wake word model and adjust the detection threshold.
  • Lower threshold = more sensitive (more false triggers). Higher = less sensitive.
  • After TTS playback, the wake word is muted for ~4 seconds to prevent echo false-triggers.

Speech-to-Text (STT)

Powered by Whisper. Choose a model size in Settings → Voice → STT Model.

  • tiny — fastest, least accurate. Good for low-power hardware.
  • base — solid balance for English. Recommended starting point.
  • small — better accuracy, moderate CPU/RAM use.
  • medium / large — highest accuracy, slowest. GPU recommended.

Set Language to your language code (e.g. en, fr) or leave blank for auto-detect. Auto-detect adds a small delay per turn.

Text-to-Speech (TTS)

Choose an engine in Settings → Voice → TTS Engine. Hot-swap without restart.

  • Kokoro — default, CPU-friendly, good quality. No extras needed.
  • Chatterbox — higher quality, more resource-intensive.
  • Dramabox — premium option. Requires: pip install -e ".[dramabox]"

Pipeline Modes (Studio)

Studio mode adjusts how the pipeline handles resources when you're recording or mixing.

  • cpu_stt — forces STT onto CPU, freeing the GPU for LLM/TTS during studio sessions.
  • pause — pauses the voice pipeline entirely during studio recording.

Configure in Settings → Voice → Studio Mode.

Silence & Timing

  • Silence Timeout — seconds of silence before the pipeline considers speech done and starts processing.
  • Silence Chunks Threshold — number of silent audio chunks before cutting off. Increase if responses get cut off mid-sentence.

Microphone Tips

  • Audio is captured as int16 (avoids PipeWire float32 normalization issues on Linux).
  • If the mic isn't detected on startup, the dashboard and text chat still work — voice is optional.
  • Check Settings → System → Pipeline Status to confirm voice pipeline state.

Web — Search Setup & Usage

Choosing a Provider

Configure in Settings → Web → Provider. Three options:

  • DuckDuckGo — default. No API key needed. Uses DuckDuckGo Instant Answers. Best for quick lookups with no setup.
  • Google — requires a Google Custom Search API key and a Search Engine ID (CX). More comprehensive results for research queries.
  • Playwright — headless browser scraping. Useful for pages that block simple HTTP requests. Requires: pip install -e ".[playwright]"

Setting Up Google Search

  1. Create a project in Google Cloud Console and enable the Custom Search API.
  2. Generate an API key and paste it into Settings → Web → API Key.
  3. Create a Programmable Search Engine at programmablesearchengine.google.com. Copy the CX (Search Engine ID) into Settings → Web → CX.
  4. Click Apply. Plia will use Google for all web searches.

Triggering Web Search in Chat

Say or type phrases like:

  • "Search for the latest news on…"
  • "Look up how to…"
  • "Find information about…"

The web agent runs, fetches results, and includes them in the response.

Max Results

Controls how many search results are passed to the LLM per query. More results = better coverage but longer processing. Default is 3. Adjust in config if needed.

LLM — Language Model Configuration

Local Model (Ollama)

Plia-OS uses Ollama as its primary LLM backend by default. No configuration needed here — set the Ollama URL and model name in the main config file (~/.plia/config.json) or via the API. Ollama must be running before starting Plia.

Fallback Provider

Configure in Settings → LLM → Provider. Used when the local model is unavailable or you want cloud inference.

  • None — local only (default). No cloud calls.
  • OpenAI — requires an OpenAI API key (sk-…). Set Model (e.g. gpt-4o). Requires: pip install -e ".[fallback-openai]"
  • Anthropic — requires an Anthropic API key. Set Model (e.g. claude-sonnet-4-6). Requires: pip install -e ".[fallback-anthropic]"

AirLLM — Large Models on Limited VRAM

AirLLM runs huge models (70B+) by loading one layer at a time, requiring only ~4 GB VRAM instead of the full model size.

  • Model — HuggingFace repo ID (e.g. meta-llama/Llama-3-70B-Instruct) or local path.
  • Compression:
    • ● 4-bit — least VRAM, fastest load, slight quality loss.
    • ● 8-bit — balanced quality and VRAM use.
    • ● None — full precision, highest quality, most VRAM.

Click Apply after setting the model. AirLLM takes priority over Ollama when configured.

System Prompt

The system prompt defines Plia's personality and behavior rules. Edit it directly in Settings → LLM → System Prompt.

  • Save — applies the new prompt immediately.
  • Undo — reverts to the previous prompt (one level of history).
  • Reset — restores the factory default prompt.

Agents — How Routing Works

Plia-OS uses a supervisor + specialist architecture. Every message goes through the supervisor, which routes to one specialist agent, then back to the supervisor to form the final response.

Routing Logic

The supervisor first tries a fast keyword match. If no keyword matches, it asks the LLM to classify the intent into one of:

  • memory — storing or recalling facts ("remember that…", "what do you know about…")
  • web — searching the internet ("search for…", "look it up…")
  • code — running Python code ("run this code…", or a fenced code block)
  • calendar — scheduling ("add to calendar…", "schedule a…")
  • home — Home Assistant control ("turn on the lights…", "set thermostat…")
  • reminder — setting reminders or timers ("remind me…", "set a timer for…")
  • respond — general conversation; no specialist needed

Agent Badges

The badges in Settings → Agents show which specialist handled the last turn. The top-bar badge also shows the active agent during processing. A badge lights up blue when active.

Agent Log

The log below the badges shows recent routing decisions — useful for debugging unexpected responses. If a message went to the wrong agent, rephrase using the trigger keywords above.

System — Pipeline & Diagnostics

Voice Pipeline States

The pipeline cycles through these states automatically:

  • armed — waiting for the wake word. Dot is dim.
  • listening — wake word detected, recording speech.
  • processing — transcribing and sending to LLM.
  • speaking — TTS is playing the response.

Use the Stop / Start button to pause the pipeline without shutting down Plia-OS. Useful when you need silence without closing the app.

System Stats

Shows live CPU, RAM, GPU, and disk usage. Click ↻ Refresh to update. Stats also appear in the top bar and update automatically every few seconds.

VRAM Bar (Top Bar)

The VRAM indicator shows GPU memory usage. Click it to expand and see which models are loaded. The Done button releases the studio model from VRAM when you're finished recording.

Config Backup

  • Export — downloads your current config as a JSON file. Back up before changing models or system prompts.
  • Import — restores config from a previously exported JSON file. Takes effect immediately.

Config is stored at ~/.plia/config.json. The PLIA_CONFIG_FILE environment variable overrides this path.

Reminders & Timers

Creating a Reminder

Two ways to create reminders:

  • Via chat — say "Remind me to take my meds at 8pm" or "Set a timer for 20 minutes." The reminder agent parses the time via the LLM and schedules it automatically.
  • Via Settings → Reminders — type the message and pick a date/time with the datetime picker, then click Add Reminder.

How Reminders Fire

A background loop checks for overdue reminders every 30 seconds. When a reminder fires:

  • It appears in the chat as a system message.
  • Plia announces it via TTS (queued between conversation turns).
  • It's marked done automatically and pruned after 7 days.

Timers vs Reminders

  • Reminders — tied to a specific date and time. "Remind me on Friday at 3pm…"
  • Timers — relative duration from now. "Set a timer for 10 minutes." Shown separately in the Timers list.

Managing Pending Items

The Pending reminders and Active timers lists show items that haven't fired yet. Click ↻ Refresh to reload. Items disappear from the list once they fire.

Calendar — Events & Google Calendar

Creating an Event

Two ways:

  • Via chat — "Add a meeting called Team Sync on Thursday at 2pm for 1 hour."
  • Via Settings → Calendar — fill in the title, date, time, and duration (minutes), then click Add Event.

Connecting Google Calendar

  1. Go to Google Cloud Console → APIs & Services → Credentials.
  2. Create an OAuth 2.0 Client ID for a Desktop app. Download the credentials.json file.
  3. In Settings → Calendar → Google Calendar, paste the full path to your credentials.json file.
  4. Click Connect Google Calendar. A browser window will open for OAuth authorization. Approve access.
  5. Once connected, the status badge shows Connected. Events created via chat or the form are synced to your Google Calendar.

Calendar ID

By default, events go to your primary calendar. To target a specific calendar, set gcal_calendar_id in ~/.plia/config.json to the calendar's ID (visible in Google Calendar settings).

Upcoming Events

The events list shows your next upcoming Google Calendar events. Click ↻ Refresh to reload. Events are read-only in this view — edit them directly in Google Calendar.

Memory — Notes & Stored Facts

Notes

Notes are quick freeform text entries stored locally. Add them via the input field or by saying "Make a note that…" in chat. Filter with the search box. Delete individual notes with the ✕ button.

Stored Memories (Facts)

Facts are key-value pairs the LLM can recall during conversation.

  • Via chat — say "Remember that my cat's name is Luna." Plia stores cat_name → Luna automatically.
  • Via Settings → Memory — enter a Key and Value manually, then click +.
  • Recall by asking "What's my cat's name?" or "What do you know about my cat?"

Chat Archive

When chat history is cleared, it's archived as a snapshot in memory under a timestamped key (e.g. chat-archive-2026-06-17T…). These appear in the Stored Memories list and can be deleted individually.

Storage Locations

  • Facts & chat history — SQLite at ~/.plia/memory.db
  • Semantic recall — ChromaDB (optional). If not installed, semantic search returns empty but fact storage still works.
  • Active chat history — separate SQLite at data/chat_history.db (project-local). Preloads last 20 turns on startup.

Memory Directory

Default: ~/.plia/. Override by setting memory_dir in ~/.plia/config.json.

Modules — Tools & Extensions

What Modules Are

Modules are Python files in the modules/ directory. Each file registers one or more tools the LLM can call during a conversation turn. Tools are auto-discovered at startup — no manual registration needed.

Adding a Custom Tool

  1. Create a .py file in the modules/ directory.
  2. Import the tool decorator: from core.registry import tool
  3. Decorate your function: @tool("Description of what this tool does")
  4. Add type hints — the decorator auto-generates the JSON schema for the LLM from them.
  5. Restart Plia-OS. The new tool appears in Settings → Modules.

Disabling a Module

Click Disable next to a module in Settings → Modules. The module is added to the disabled_modules list in config and will be skipped on next startup. Click Enable to re-enable it. Changes take effect after restart.

Module List

The list shows all currently loaded modules with their filenames and registered tool names. Click ↻ Refresh to reload the list. A module showing as loaded means all its tools are available to the LLM right now.

Home — Home Assistant Integration

Setup

  1. In Home Assistant, go to Profile → Long-Lived Access Tokens and create a new token.
  2. In Settings → Home, enter your Home Assistant URL (e.g. http://homeassistant.local:8123) and paste the token.
  3. Click Save. Plia connects immediately — no restart needed.

Controlling Devices via Chat

Use natural language to control entities:

  • "Turn on the living room lights."
  • "Set the thermostat to 22 degrees."
  • "Turn off all the lights."
  • "What's the temperature in the bedroom?"

The home agent translates your request into Home Assistant service calls using the entity list it fetches from your HA instance.

Entity Discovery

Plia fetches your Home Assistant entities automatically when the home agent is triggered. The Settings → Home panel shows a list of discovered entities for reference. Click ↻ Refresh to reload after adding new devices to HA.

Supported Entity Types

  • Lights (light.*) — on/off, brightness, color
  • Switches (switch.*) — on/off
  • Climate (climate.*) — temperature, mode
  • Sensors (sensor.*) — read current values
  • Scripts & automations — trigger by name

FAQ

The top bar shows "error" — what's wrong?

The status badge shows the voice pipeline state. "error" means the pipeline failed to start — usually no microphone detected, a missing audio dependency, or an STT model that couldn't load. Text chat still works normally. Check Settings → System → Pipeline Status and the terminal log for details.

Voice isn't working — no response after wake word.

Check in order: (1) pipeline is running — Settings → System dot should be green; (2) wake word threshold isn't too high — lower it in Settings → Voice; (3) STT model loaded — terminal log will show errors; (4) microphone permissions — Plia needs access to the default input device.

How do I change the wake word?

Go to Settings → Voice → Wake Word Model. Select a different openwakeword model from the dropdown and click Apply. The pipeline reloads the detector automatically.

Responses are slow. How do I speed things up?

  • Switch to a smaller STT model (tiny or base) in Settings → Voice.
  • Use a faster/smaller Ollama model — edit ollama_model in ~/.plia/config.json.
  • Switch TTS to Kokoro (lightest engine) in Settings → Voice.
  • If using AirLLM, switch compression to 4-bit in Settings → LLM.

Can I run Plia-OS without a GPU?

Yes. Kokoro TTS and Whisper STT both run on CPU. Ollama also runs CPU-only (slower). AirLLM requires a GPU. Set a small Ollama model (e.g. llama3.2:3b) for best CPU performance.

What's the difference between Notes and Memories?

  • Notes — freeform text you add manually. Not used by the LLM during conversation.
  • Memories (facts) — key/value pairs the LLM can recall mid-conversation. Say "remember that…" or add via Settings → Memory.

How do I reset the system prompt to default?

Go to Settings → LLM → System Prompt and click Reset. To undo a single change without full reset, click Undo instead.

How do I add a new tool or capability?

Create a .py file in modules/, decorate a function with @tool("description") from core.registry, and restart Plia-OS. The tool appears automatically. See Docs → Modules for details.

How do I disable a module without deleting it?

Go to Settings → Modules, find the module, and click Disable. It's added to disabled_modules in config and skipped on next restart. Re-enable the same way.

Where is my data stored?

  • Config — ~/.plia/config.json
  • Facts & memories — ~/.plia/memory.db
  • Active chat history — data/chat_history.db (project directory)
  • Reminders — ~/.plia/memory.db (reminders table)

Back up via Settings → System → Export Config, or copy the files above directly.

How do I clear the conversation history?

Click Clear Chat in the bottom bar. This archives the current session to memory (as a timestamped key) and starts fresh. The voice pipeline's in-memory context also resets.

Can I use a cloud LLM instead of Ollama?

Yes. Install the extra (pip install -e ".[fallback-openai]" or ".[fallback-anthropic]"), then set Provider, Model, and API Key in Settings → LLM → Fallback Provider and click Apply.

How do I back up and restore my full config?

Go to Settings → System → Config Backup. Click Export to download a JSON file. Click Import to restore from a previously exported file. Takes effect immediately.
